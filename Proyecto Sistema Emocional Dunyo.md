Aquí está la parte importante.

No vamos a medir “cómo se siente” a secas. Vamos a medir:

7.1 Emotional Load Score (ELS) Carga emocional general del niño. Se alimenta de:

● frecuencia de eventos

● intensidad

● diversidad emocional

● repetición en pocos días

Escala

0–100

7.2 Relationship Emotional Weight (REW) Peso emocional por persona.

Mide:

● con quién aparecen más emociones

● intensidad media por persona

● frecuencia de repetición

● tipo de emoción ligada a esa persona

Ejemplo

● mamá: 62

● papá: 34

● amigo Mateo: 71

7.3 Context Pressure Score (CPS) Presión emocional por contexto. Contextos

● casa

● escuela

● cuarto

● amigos

● digital / chat / juego

● trayecto

● otro

7.4 Intensity Trend Index (ITI)

Tendencia de intensidad en ventana móvil de tiempo. Objetivo

Ver si:

● está subiendo

● se mantiene

● baja

● se dispara por persona o contexto

7.5 Recovery Speed Score (RSS) Velocidad de recuperación.

Mide

Tiempo entre registro de evento y sensación posterior más regulada.

7.6 Communication Intent Score (CIS) Intención de comunicar.

Se alimenta de:

● cuántas veces elige hablar

● con quién quiere hablar

● cuántas veces evita

● cuántas veces pospone

7.7 Suppression Index (SI) Índice de supresión o evitación. Mide Veces que:

● registra emoción

● pero elige no hacer nada

● o no hablar

● o guardar sin actuar

7.8 Emotional Volatility Index (EVI) Volatilidad emocional.

Mide

Saltos bruscos entre emociones/intensidades en corto plazo.

7.9 Trust Distribution Map (TDM) Mapa de confianza.

Mide

A quién sí recurre, a quién no, y con quién asocia calma vs carga.

7.10 Help Utility Score (HUS)

Qué tanto ayudan las herramientas. Se alimenta de:

● acción elegida

● respuesta posterior

● “me ayudó / no me ayudó”

● recurrencia

8) Reglas de negocio para métricas Para cada evento emocional registrar:

● emoción principal

● emoción secundaria opcional

● intensidad

● persona relacionada

● contexto/lugar

● fecha/hora

● acción elegida

● si se comunicó o no

● resultado posterior

Regla de agrupación

● ventanas de 24h

● ventanas de 7 días

Regla de relevancia

Una sola emoción aislada no genera alerta. Patrón + intensidad + repetición sí.

10.1 APIs obligatorias para empezar

1. Supabase

Lo necesitas para:

Supabase documenta Auth, Database, Storage y Edge Functions como piezas nativas de la plataforma, y las secrets para funciones se gestionan desde el dashboard o CLI.

2. Resend

Lo necesitas para:

● correos semanales a padres ● correos de alertas

● plantillas de correo

Resend soporta API de envío, templates hospedados y variables para plantillas.

3. Expo Notifications

Lo necesitas para:

● push notifications

● alertas al padre

● nudges suaves al niño

● recordatorios de uso

Expo documenta expo-notifications para obtener tokens, programar, recibir y responder notificaciones; además su servicio de push integra con FCM/APNs y usa credenciales de Android e iOS.

4. OpenAI API

Lo necesitas para:

● clasificación emocional estructurada

● resúmenes parentales

● recomendaciones textuales

● generación de mensajes “hablar con ___”

● motor de insights

OpenAI hoy centra estas capacidades en la Responses API, con soporte para structured output y function calling en la documentación general; además hay modelos más rápidos/económicos y modelos de razonamiento más pesados si luego los necesitas.

13.1 Clasificación emocional

Objetivo: convertir entradas cortas en estructura usable.

Prompt base:

“Eres un motor de clasificación emocional para una app de niños de 8 a 12 años. No diagnostiques ni uses lenguaje clínico. Recibirás emoción, intensidad, persona relacionada, contexto y acción elegida. Devuelve JSON con: emotion_primary, emotion_secondary, load_level, probable_pattern, relationship_weight_hint, context_pressure_hint, suggested_action, parent_summary_safe.”

13.2 Recomendador de acción

Objetivo: dar ayuda breve.

Prompt base:

“Genera una acción breve, clara y usable en menos de 2 minutos para un niño de 8 a 12 años según emoción, intensidad, persona relacionada y contexto. No uses lenguaje médico, moralista ni infantilón. Devuelve: title, 3 steps max, duration, tone, completion_goal.”

13.3 Resumen parental

Objetivo: explicar sin invadir.

Prompt base:

“Resume patrones emocionales de un niño para su padre o madre. No diagnostiques ni alarmes. No reveles detalles íntimos textuales. Enfócate en tendencias, intensidad, personas involucradas, contextos y sugerencias de conversación. Devuelve: weekly_summary, key_patterns, people_involved, contexts, what_changed, suggested_conversation, what_to_avoid, escalation_needed.”

13.4 Mensaje hacia persona importante

Objetivo: ayudarle a hablar con alguien.

Prompt base:

“Convierte emoción + intensidad + persona elegida + objetivo en un mensaje simple, natural y directo que un niño de 8 a 12 años podría usar sin pena. Mantén máximo 2 líneas. Devuelve 3 variantes: suave, directa, muy simple.”
